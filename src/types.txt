// This file defines types of Chinese characters in unicode and relationships between them.
//
// The format of a relationship:
// [unicode] | [type] (| [altunicode])?
//
// [unicode] and [altunicode] should be unicode code points.
//
// [type] should be one of the following:
//
// [type] 0 indicates that the unicode is generic to both the simplified and traditional character sets.
// example: 4e00 | 0
//
// [type] 1 indicates that the [unicode] on the left is a simplified form of the [altunicode] traditional form on the right.
// example: 6c49 | 1 | 6f22
//
// [type] 2 indicates that the [unicode] on the left is a traditional form of the [altunicode] simplified form on the right.
// example: 6f22 | 2 | 6c49
//
// [type] 3 indicates that the unicode is an equivalent form to the [altunicode] on the right.
// example: 8aac | 3 | 8aaa
//
// Notes:
//
// Each unicode appearing on the right of a relationship should be defined elsewhere on the left of another.
// 
// There is not any restriction on the degree of relationships, though it is generally the case that a 
// traditional form should uniquely map to a simplified form, however.  The reverse is not true, of course.
//
// This is an amateur effort and should not be used for anything very important.
